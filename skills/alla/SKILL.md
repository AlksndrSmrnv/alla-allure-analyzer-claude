# alla — триаж упавших тестов Allure TestOps

`alla` — сервис автоматического анализа упавших автотестов: кластеризует
падения по корневой причине, ищет совпадения в базе знаний, прогоняет
LLM-анализ через GigaChat, генерирует HTML-отчёт.

Этот skill объясняет, как агенту вызывать `alla` через её **MCP-сервер**.

## Когда использовать

Запускай инструменты `alla` если пользователь:

- называет ID запуска и просит «проанализируй прогон 12345»;
- спрашивает «почему упал launch …», «дай отчёт по падениям»;
- просит выгрузить кластеры падений / совпадения с базой знаний;
- хочет HTML-отчёт по конкретному прогону.

## Доступные MCP-инструменты

MCP-сервер `alla` выставляет два инструмента (полные сигнатуры
прилетают агенту автоматически из `tools/list`):

| Инструмент | Когда вызывать |
|-----------|----------------|
| `analyze_launch(launch_id, push_to_testops?)` | Базовый анализ. Возвращает компактный JSON: счётчики падений, кластеры (label/size/signature), совпадения базы знаний, краткий LLM-вердикт. |
| `analyze_launch_html(launch_id, push_to_testops?)` | То же + сохраняет HTML-отчёт. Возвращает `report_url` (если хранилище отчётов настроено) и компактную сводку. |

**Параметр `push_to_testops`** — переопределяет `ALLURE_PUSH_TO_TESTOPS`
для одного вызова. По умолчанию (None) используется значение из конфига
сервера. Передавай `false`, когда пользователь просит «только посмотреть,
не писать в TestOps».

## Подключение MCP-сервера

`alla` поднимает MCP по тому же порту, что и REST API — endpoint
`http://<host>:8090/mcp` (streamable HTTP). Запустить сервер:

```bash
pip install -e .  # из корня репозитория alla
alla-server       # слушает на 0.0.0.0:8090
```

Минимально нужны переменные окружения (`.env` рядом):

```env
ALLURE_ENDPOINT=https://allure.example.com
ALLURE_TOKEN=...
ALLURE_PROJECT_ID=1
```

### qwen CLI

В `~/.qwen/settings.json` добавь блок `mcpServers`:

```json
{
  "mcpServers": {
    "alla": {
      "type": "http",
      "url": "http://localhost:8090/mcp"
    }
  }
}
```

После рестарта qwen CLI проверь, что инструменты `analyze_launch` и
`analyze_launch_html` появились в списке доступных tools.

## Правила вывода

- В пользовательских ответах пиши **«база знаний»**, не «KB».
- Сохраняй язык ответа русским (как у пользователя).
- Если `analyze_launch_html` вернул `hint` — пробрось его пользователю,
  не пытайся «исправить» автоматически.
- Если в ответе есть `report_url` — сделай ссылку кликабельной.

## Если что-то пошло не так

- Ошибка `alla 401` — проверь `ALLURE_TOKEN` на стороне `alla-server`.
- Ошибка `alla 404` — такой launch не существует в проекте.
- Ошибка `alla 502` — TestOps недоступен / отдал не-2xx.
- Сервер не отвечает на `/mcp` — убедись что `alla-server` запущен
  (`curl http://localhost:8090/health` должен вернуть `{"status":"ok","mcp":true}`).
