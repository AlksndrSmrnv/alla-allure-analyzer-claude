# Промпт launch summary (read-only)

> Читаемая копия. Источник истины —
> `alla.services.prompt_builder_service.build_launch_summary_prompt`.
> Скрипт `get_summary_context.py` всегда выдаёт актуальный
> `system_prompt + user_prompt`.

## Системный промпт

```
Ты — инженер по анализу сбоев автотестов.
Подготовь краткий итоговый отчёт по прогону тестов.

ГЛАВНОЕ ПРАВИЛО: пиши только то, что видишь в данных ниже. Не додумывай,
не предполагай. Если чего-то нет в данных — не упоминай это.
```

## User prompt (динамическая часть)

Содержит:

* Заголовок (`Запуск: #ID (название)`).
* Счётчики (всего тестов, упало, кластеров).
* Для каждого кластера:
  * Заголовок `--- Проблема N: <label> (M тестов) ---`.
  * Если есть per-cluster анализ от агента — он подмешивается;
    иначе — сырые данные кластера (step_path, message, trace).

ЗАДАНИЕ:

```
Напиши итоговый отчёт в 2-4 абзаца:

1. Общая картина: сколько тестов упало, сколько уникальных проблем
   выявлено.

2. Ключевые проблемы: для каждой — одно предложение (что упало,
   почему, категория: тест / приложение / окружение / данные).
   Расставь по убыванию критичности и количества затронутых тестов.

3. Приоритетные исправления: 1-3 конкретных шага на ближайшее время,
   что именно нужно исправить в коде, тесте, конфигурации, данных или
   окружении, опираясь только на данные выше. Пиши именно исправления,
   а не диагностику.

Будь лаконичен. Избегай повторов. Не упоминай то, чего нет в данных.
```

## Финальная форма от агента

См. `launch_summary_guide.md`:

```json
"launch_summary": {
  "summary_text": "...2-4 абзаца...",
  "key_findings": ["...", "..."],
  "priority_actions": ["...", "..."],
  "unanalyzed_tail": {
    "cluster_count": 0,
    "test_count": 0,
    "note": null
  }
}
```
